Maze Stack Game

This project was created as a final for CS 301 and centers around a stack-based game implemented entirely in C++. The player explores a maze composed of multiple rooms, using vertical navigation through stacks and horizontal transitions through doorways.

The game runs in a text-based console. You can move up or down using keys 'w' and 's', enter doors using 'x', grab a treasure with 'z', and quit the game with 'q'. The stack structure provides a great way to simulate depth, such as tiles inside a room being accessed one layer at a time.

The logic behind the maze is structured using the Standard Template Library (STL) stack. This allows the tiles to be pushed or popped efficiently, giving us natural room traversal and reset behaviors. Each room is essentially a vertical column, and your pointer navigates up or down it like a memory stack.

We also use vectors to temporarily reverse the stack for display purposes, and simple pointer arithmetic to track the current tile the player is on. Rather than actual memory addresses, we use indices as abstracted pointers to simplify logic and avoid direct memory manipulation.

The project demonstrates a solid understanding of stack operations, pointer simulation, modular logic with functions, and interactive loop structures — all important foundations in computer engineering and systems programming.

16-BIT MIPS Design

This 16-bit MIPS processor is a simplified architecture designed to demonstrate core CPU operations such as register access, arithmetic processing, and instruction execution. Each component works together to simulate real-world instruction flow in a modular, scalable way.

Select the tabs above to dive deeper into the specific architectural blocks, including the Arithmetic Logic Unit (ALU) and the Register File.

Arithmetic Logic Unit (ALU)

An Arithmetic Logic Unit (ALU) is a core component of the CPU responsible for executing operations like addition, subtraction, bitwise AND/OR/XOR, and comparisons. It uses input from the register file and returns results back into memory or registers.

Register File

A register file is a small, fast memory unit within a CPU that shifts temporary data needed during instruction execution. Unlike RAM, the register file offers quick access to values used in arithmetic, logic, and data movement operations.

Each register is built with flip-flops and stores a fixed-width word (e.g., 8 bits). Flip-flops are synchronized with a clock and controlled by enable and reset lines.
